Indotherm: a full theme with all the trimmings

Indotherm is a set of files that control the look of the Grawlix CMS. Collectively called a “theme,” these files demonstrate what how you can make a webcomic look.

Who should use this theme

Indotherm is a fully-developed theme, complete with a built-in grid, icons, CSS reset, and typography. It uses several thousand lines of CSS to cover many, many possible browser configurations and reader scenarios. If you’re not comfortable with HTML and CSS, or want an easier introduction to theming with the Grawlix CMS, check out the Stick Figure and Asterisk themes instead of Indotherm.

Customizing this theme

Many artists want to change a few things like:

  • The site’s logo
  • The site’s background
  • Link and text colors
  • Site width

The site logo

  1. Upload your logo image to /themes/indotherm (the same folder in which you found this file).
  2. Open the file “snippet.header.php”.
  3. Replace the image src on line 32 with the path to your image. Don’t forget to use a sensible alt attribute.

The site’s background

There are two ways to change a site’s background color and image. Start by changing the parameters on line 210 in theme.css. Then open the tone files (e.g. “tone.whiteout.css”) to make more specific changes — see Themes and Tones below.

Text and link colors

They’re in theme.css on lines 211 and 435, respectively. Tone files may override these colors, however — see Themes and Tones below.

Site width

Not everyone has the same screen as you, or views your site with their browser open as wide as possible. Comics that are, say, 1200 pixels wide must shrink to fit screens that display less than 1200 pixels across. So two factors influence your site’s layout: your comic image’s size and the reader’s browser size.

Found on line 3055 (for the comic image) and line 2798 (for the site), Indotherm’s media queries control page and image widths on differently-sized browsers. In plain English they say, “if the screen is up to X pixels wide, then make the element no more than Y pixels wide.”

Themes and tones

Indotherm’s “theme.css” file controls the site’s general look: its layout, typography and broad colors. Tone files, whose names begin with “tone” and end with “css”, modify that look. For example, the theme.css file might say “make links blue,” but a tone file says, “no, make them orange.”

Here’s why: The theme.css file applies to your whole site. Tone files, though, you can apply to individual chapters or static pages. That means you could, say, make your FAQ and Cast pages greyscale, but your comic in full color. Or if your story changes from winter to summer in two chapters, each can have its own motif.

Assign themes and tones to your site in “Themes” of your admin panel.

PHP functions

Snippet()

Snippets are PHP files that (surprise) begin with “snippet” and end with “php”. The snippet function grabs such a file and sticks it into place. For example, snippet('footer') adds the file “snippet.footer.php”. Check out the docs for details.

Show()

“Show” functions grab bits of data from the admin panel. For example, show('artist_name') inserts the name you provide in Settings of your admin panel. See the docs for more info.

PHP scripts

So, what’s in the box? Every Grawlix theme comes with an assortment of PHP scripts that generate web pages that the reader ultimately sees. Each script is part of a whole page.

Page.archive

Displays links to your comic pages, in order. Control its appearance (what the links’ text is) in the admin panel.

Page.comic

Displays comic images — the “comic pages” themselves — as well as blog post snippet, transcript snippet, sharing snippet, etc.

Page.static

Displays content for non-comic pages like the FAQ and “About” page.

Snippet.archive-nav

Comic navigation links that includes one to the archives page.

Snippet.comic-nav

The first/previous/next/latest links in your list of comic pages.

Snippet.comic-nav-random

Navigation links … but includes a link to a certain comic page. Which page? Who knows? It’s different every time, giving readers a surprise at every page load.

Snippet.comments

Contains code that enables readers to leave their thoughts on your comic pages — Disqus, etc. Kinda complicated here … tinker with this only if you get updated code from your commenting service of choice.

Snippet.follow

Contains links to other places on the ’net people can find you, like RSS, Facebook, Pinterest and Twitter. Best left alone unless you know a little PHP.

Snippet.footer

The end of every web page on your site. This snippet includes the copyright statement and publish frequency, both of which you change in the admin panel or custom-write by simply replacing all the code in line 5.

Snippet.header

Begins each web page with info about the HTML, favicons, and — by default — site logo, title and site-wide navigation menu.

Snippet.main-menu

Displays a group of links for the site-wide navigation or menu. We left this file deliberately simple in case you want to make it more complex with, for example, hard-coded links.

Snippet.share

This complicated-looking script gives readers the chance to tell other people about your comic pages via social media. You can control which appears in the site via your admin panel’s sharing options.

Snippet.twitterstream

Puts a Twitter link into your comic. Best left alone unless you know PHP or have your own Twitter link code; enter your Twitter info in the admin panel to make this snippet work.

Snippet.transcript

Shows a comic page’s written dialog, events, setting, etc to the reader.

Pattern.(something).php

Pattern files are bits of HTML that structure static pages’ content blocks. You can create your own pattern files or edit the included ones to fit your needs.
